# BuildCorp - 1차 기획 논의 결과

## 한 줄 요약
> 도트 아트 가상 사무실에서 AI 직원들이 당신의 실제 업무를 수행하는 데스크톱 앱

---

## 정체성
- 게임이 아닌 **실제 업무 도구** (게임적 요소로 시각화)
- 도트 디자인 캐릭터 + HD UI 혼합 스타일
- 데스크톱 앱 (Electron)

## 핵심 원칙
1. **실용성 우선** - 게임 재미보다 실제 업무 수행 능력이 핵심
2. **시각적 즐거움** - 도트 캐릭터들이 일하는 모습으로 진행 상황을 직관적으로 파악
3. **유연한 구조** - 어떤 업종이든 자유롭게 회사를 설정하고 팀을 구성
4. **협업 생태계** - 내 여러 회사 간, 다른 사용자 간, 외부 서비스와 연동

---

## 회사 구조

- 사용자가 자유롭게 회사 이름, 업종, 설명 설정
- 한 사용자가 여러 회사(프로젝트) 운영 가능
- 회사 내 팀 자유 생성 (개발팀, 마케팅팀, 디자인팀 등)
- 팀 내 AI 에이전트(직원) 배치
- 각 회사는 독립적인 팀 구조 보유

## AI 에이전트 시스템

### 다중 LLM 지원
- Claude API (Anthropic)
- OpenAI API (GPT-4.1, GPT-4o 등)
- Ollama (로컬 모델)
- *추후 추가: 셀레니움 기반 ChatGPT 웹 자동화 프로바이더*

### 에이전트별 커스터마이징
- 이름, 도트 아바타
- 전문 분야, 성격/톤 설정
- System Prompt 커스터마이징
- LLM 프로바이더/모델 개별 선택

### 에이전트 도구 시스템 (Tool Use)
에이전트가 실제 작업을 수행하기 위한 도구. 파일을 읽고 쓰는 건 LLM이 직접 하는 게 아니라, **BuildCorp 앱이 중간에서 실행**하고 결과를 LLM에 전달하는 구조.

| 기본 도구 | 설명 |
|-----------|------|
| `readFile` | 특정 파일 읽기 |
| `writeFile` | 파일 생성/수정 |
| `listDir` | 폴더 구조 보기 |
| `search` | 파일 내 텍스트 검색 |
| `runCommand` | 터미널 명령 실행 (빌드, 테스트 등) |

추가 도구는 MCP를 통해 확장 가능 (아래 MCP 섹션 참조).

### 파일 탐색 전략
파일이 많은 프로젝트를 다룰 때 LLM 컨텍스트 한계 대응:
1. **폴더 구조만 먼저 전달** (파일 목록, 트리)
2. **LLM이 필요한 파일 선택** → 해당 파일만 읽기
3. **추가로 필요한 파일 요청** → 반복
4. 사람이 코드 리뷰하는 것과 동일한 방식

### 에이전트 통계
- 업무 완료 횟수/시간 트래킹 (가벼운 수준)
- 분야별 작업 횟수 기록

## 병렬 실행 & 워크플로우

### 병렬 실행
- 모든 에이전트가 동시에 독립적으로 작업 가능
- 각 에이전트는 자신의 async 컨텍스트에서 실행

### 워크플로우 에디터 (직원 카드 기반 비주얼 노드 에디터)
에이전트 간 연계 작업을 **게임적 UI로 설정**하는 시스템.

#### 카드 구성
```
┌─────────┐
│  ⌨️ 민수  │
│  개발팀   │
│ "코드작성"│
│  🟢 작업중│
└─────────┘
```
- 도트 아바타 + 이름
- 소속 팀
- 이 워크플로우에서 맡은 역할/태스크
- 현재 상태 (대기/작업중/완료)

#### 연결 방식
카드에서 카드로 드래그하여 연결선 생성. 각 연결마다 전환 조건 설정:
- **자동**: AI가 판단해서 바로 다음 단계로
- **승인 필요**: 사용자가 결과 확인 후 넘김
- **조건부**: 특정 조건 만족 시 자동, 아니면 승인 요청

#### 지원 패턴
- `A → B → C` 순차 체인
- `A → (B, C)` 병렬 분기
- `(B, C) → A` 합류 (결과 종합)
- 자유 조합

#### 워크플로우 관리
- **템플릿**: 완성된 워크플로우를 저장, 에이전트만 교체해서 재사용
- **즉석 구성**: 매번 태스크마다 새로 만들기 가능
- **실행 중 수정**: 아직 시작 안 된 단계는 추가/변경/삭제 가능, 완료된 단계는 수정 불가

## 인벤토리 시스템 (도구 관리)

에이전트에게 부여하는 도구를 **RPG 인벤토리** 방식으로 관리.

```
[장착 중]     → 에이전트가 사용 가능한 도구
[보관함]      → 설치했지만 이 에이전트에겐 미배정
[MCP 스토어]  → 새 도구 검색/설치
```

- 도트 아이콘으로 도구 표현
- 드래그로 장착/해제
- 에이전트마다 개별 설정
  - 개발 에이전트: readFile, writeFile, GitHub, 터미널 전부 장착
  - 검토 에이전트: readFile, search만 장착 (쓰기 권한 없음)

## MCP 연동

### BuildCorp → MCP 서버 (도구 가져다 쓰기)
- MCP 스토어에서 서버 검색/설치 (Smithery, mcp.so 등 레지스트리 연동)
- 설치된 MCP 도구를 에이전트 인벤토리에서 장착/해제
- GitHub, Slack, Notion, DB 등 외부 서비스와 연동

### BuildCorp ← MCP 클라이언트 (외부에서 접속)
- BuildCorp 자체를 MCP 서버로 노출
- Claude Code 등에서 BuildCorp에 접속하여 태스크 생성/결과 조회 가능

---

## 업무 방식 (3가지)

### 1. 채팅
- 에이전트 클릭 → 채팅창 열림 → 대화로 업무 지시
- 실시간 스트리밍 응답
- 파일 첨부 지원
- 간단한 업무에 적합

### 2. 태스크 보드 (칸반)
- Todo → In Progress → Review → Done
- 태스크 카드에 상세 설명, 첨부파일, 기한 설정
- 에이전트에게 드래그&드롭으로 배정
- 여러 에이전트가 하나의 태스크에 참여 가능

### 3. 워크플로우
- 위 "워크플로우 에디터" 섹션 참조
- 복잡한 다단계 업무에 적합

## 화면 구성 (4개 뷰)

### 1. 사무실 뷰 (도트 아트)
- PixiJS 캔버스로 렌더링
- 팀별 구역에 도트 캐릭터 배치
- 상태별 애니메이션 (일하기, 대기, 회의, 휴식)
- 워크플로우 진행 시 직원 간 서류 전달 애니메이션
- 캐릭터 클릭 → 채팅 패널 열림

### 2. 대시보드 뷰 (HD UI)
- 회사 현황 카드 (활성 태스크, 완료율, 팀 상태)
- 에이전트 성과 통계
- 재정 현황

### 3. 태스크 뷰
- 칸반 보드 UI
- 태스크 생성/배정/완료 관리

### 4. 워크플로우 뷰
- 직원 카드 기반 노드 에디터
- 실시간 진행 상태 상세 표시
- 템플릿 저장/불러오기

### 메인 레이아웃
```

│  [회사명 ▼]  [사무실] [대시보드] [태스크] [워크플로우] [설정]  │
├──────────┬──────────────────────────────────┤
│          │                                  │
│  팀 목록  │         메인 콘텐츠 영역           │
│  에이전트 │                                  │
│  트리     │                                  │
│          │                                  │
│          ├──────────────────────────────────┤
│          │        채팅/상호작용 패널           │
├──────────┴──────────────────────────────────┤

└─────────────────────────────────────────────┘
```

## 협업

### 내 여러 회사 간
- 회사 A의 에이전트가 회사 B에 업무 요청
- 프로젝트 간 데이터/결과물 공유

### 멀티플레이어 (다른 사용자)
- WebSocket 기반 협업 서버
- "외주" 개념: 다른 회사의 에이전트에게 업무 위탁
- 결과물 주고받기

### 외부 서비스 연동
- MCP를 통한 Slack, GitHub, Notion, Google Drive 등 연결
- 커스텀 MCP 서버 추가 가능

## 기술 스택

| 레이어 | 기술 | 이유 |
|--------|------|------|
| 프레임워크 | Electron + Vite | 크로스 플랫폼 데스크톱, 빠른 HMR |
| 프론트엔드 | React + TypeScript | 컴포넌트 기반 UI |
| 픽셀 렌더링 | PixiJS | 사무실 뷰의 도트 캐릭터/애니메이션 |
| HD UI | Tailwind CSS + shadcn/ui | 대시보드/패널의 깔끔한 HD UI |
| 상태 관리 | Zustand | 가벼운 전역 상태 관리 |
| 로컬 DB | SQLite (better-sqlite3) | 회사/직원/태스크 데이터 저장 |
| AI 통합 | Multi-LLM (Claude, OpenAI, Ollama) | 다중 LLM 지원 |
| 실시간 협업 | WebSocket (Socket.io) | 멀티플레이어 회사 간 협업 |
| 외부 연동 | MCP Protocol | Slack, GitHub, Notion 등 연결 |

## 구현 단계

### Phase 1: 기반 구축 ✅
- Electron + Vite + React + TypeScript 프로젝트 셋업
- 기본 윈도우 및 레이아웃 구성
- SQLite 데이터베이스 스키마 설계 및 연결
- 회사/팀/에이전트 CRUD 기능
- 기본 라우팅 및 네비게이션

### Phase 2: AI 에이전트 코어
- LLM 프로바이더 추상화 레이어 (Claude, OpenAI, Ollama)
- 에이전트 도구 시스템 (readFile, writeFile, listDir, search, runCommand)
- 에이전트 실행 엔진 (system prompt + 도구 사용 루프)
- 채팅 인터페이스 (직원과 1:1 대화)
- 스트리밍 응답 처리
- 파일 첨부 및 결과물 저장

### Phase 3: 태스크 & 워크플로우
- 태스크 보드 (칸반) UI 완성
- 태스크 생성/배정/완료 플로우
- 워크플로우 에디터 (직원 카드 노드 에디터)
- 워크플로우 실행 엔진 (병렬/순차/분기/합류)
- 단계 전환 조건 (자동/승인/조건부)
- 워크플로우 템플릿 저장/불러오기

### Phase 4: 도트 사무실 뷰
- PixiJS 캔버스 통합
- 도트 스프라이트 시트 제작/적용
- 캐릭터 상태 애니메이션
- 사무실 레이아웃 및 가구 배치
- 워크플로우 진행 시 서류 전달 애니메이션
- 캐릭터 클릭 인터랙션

### Phase 5: 인벤토리 & MCP
- 인벤토리 UI (장착/보관함/스토어)
- MCP 클라이언트 (외부 도구 가져다 쓰기)
- MCP 서버 (외부에서 BuildCorp에 접속)
- MCP 스토어 (검색/설치)
- 에이전트별 도구 권한 관리

### Phase 6: 협업 시스템
- 내 여러 회사 간 데이터 공유
- WebSocket 기반 멀티플레이어 서버
- 회사 간 업무 위탁 기능

### Phase 7: 폴리시 & 배포
- 알림 시스템
- 설정 (API 키 관리, 테마, 단축키)
- 사무실 커스터마이징 (가구, 테마)
- 빌드 및 배포 (electron-builder)
