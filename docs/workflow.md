# 워크플로우 시스템

## 컨셉

에이전트 간 연계 작업을 **직원 카드 기반 비주얼 노드 에디터**로 설정.
딱딱한 플로우차트가 아니라, **도트 직원 카드를 배치하고 연결하는 게임적 UI**.

## 직원 카드

```
┌─────────┐
│  ⌨️ 민수  │  ← 도트 아바타 + 이름
│  개발팀   │  ← 소속 팀
│ "코드작성" │  ← 이 워크플로우에서의 역할
│  🟢 작업중 │  ← 현재 상태 (대기/작업중/완료)
└─────────┘
```

## 연결

카드에서 카드로 **드래그하여 연결선 생성**.
연결선 위에 전환 조건 뱃지 표시. 도트 스타일 점선 화살표로 게임 느낌.

## 전환 조건 (각 연결마다 사용자가 개별 설정)

| 조건 | 설명 |
|------|------|
| **자동** | AI가 판단해서 바로 다음 단계로 넘김 |
| **승인 필요** | 사용자가 결과 확인 후 수동으로 넘김 |
| **조건부** | 특정 조건 만족 시 자동, 아니면 승인 요청 |

## 지원 패턴

```
순차 체인:     A → B → C
병렬 분기:     A → (B, C) 동시 실행
합류:         (B, C) → A가 결과 종합
자유 조합:     위 패턴을 자유롭게 조합
```

### 예시: 코드 리뷰 파이프라인

```
┌─────────┐         ┌─────────┐         ┌─────────┐
│  ⌨️ 민수  │───자동──▶│  💻 지영  │──승인필요─▶│  🔍 철수  │
│  기획팀   │         │  개발팀   │         │  QA팀    │
│ "기획서"  │         │ "코드"   │         │ "리뷰"   │
│  ⏳ 대기중 │         │  🟢 작업중 │         │  💤 대기  │
└─────────┘         └─────────┘         └─────────┘
                         │
                         ├───자동──▶ ┌─────────┐
                         │          │  🧪 수진  │
                         │          │  QA팀    │
                         │          │ "테스트"  │
                         │          │  💤 대기  │
                         │          └─────────┘
```

## 워크플로우 관리

### 템플릿
- 완성된 워크플로우를 **템플릿으로 저장**
- 불러올 때 에이전트만 교체해서 재사용
- 예: "코드 리뷰 파이프라인", "콘텐츠 제작 플로우"

### 즉석 구성
- 매번 태스크마다 새로 만들기 가능
- 빈 캔버스에서 카드 배치

### 실행 중 수정
- 아직 **시작 안 된 단계**: 추가/변경/삭제 가능
- **이미 완료된 단계**: 수정 불가 (히스토리 보존)
- 예: "중간에 디자인 검토도 넣어야겠다" → 미시작 단계 사이에 끼워넣기

## 시각화 (이중)

| 뷰 | 표현 방식 |
|----|----------|
| **사무실 뷰** | 직원들이 서류를 전달하는 도트 애니메이션 (간단) |
| **워크플로우 뷰** | 카드별 실시간 상세 진행상태 (상세) |
