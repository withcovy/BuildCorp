# MCP 연동

## MCP란?

Model Context Protocol. 외부 도구/서비스를 **플러그인처럼 연결하는 규격**.
에이전트의 능력을 확장하는 수단.

## 두 가지 방향

### 1. BuildCorp → MCP 서버 (외부 도구 가져다 쓰기)

BuildCorp이 MCP 클라이언트로서 외부 MCP 서버에 연결.

```
BuildCorp 에이전트
  ├── 기본 도구 (readFile, writeFile, search...)
  └── MCP 서버 연결
      ├── GitHub → 에이전트가 직접 PR 생성, 이슈 관리
      ├── Slack → 에이전트가 메시지 전송/수신
      ├── Notion → 문서 작성/업데이트
      ├── Figma → 디자인 파일 읽기
      ├── DB 서버 → 쿼리 실행
      └── 커스텀 MCP 서버 → 뭐든 추가 가능
```

### 2. BuildCorp ← MCP 클라이언트 (외부에서 접속)

BuildCorp 자체를 MCP 서버로 노출.

```
Claude Code ──MCP 연결──▶ BuildCorp 서버
                          ├── 태스크 생성
                          ├── 에이전트에게 업무 지시
                          └── 결과 조회
```

Claude Code에서 "BuildCorp의 개발팀한테 이거 맡겨" 같은 것도 가능.

## MCP 스토어

앱 내에서 MCP 서버를 검색/설치하는 기능.
레지스트리 연동 (Smithery, mcp.so 등).

```
┌─────────────────────────────┐
│  🔌 MCP 도구 스토어          │
│                             │
│  🔍 검색: [database     ]   │
│                             │
│  ┌─────────────────────┐    │
│  │ 📁 GitHub            │    │
│  │ PR, 이슈, 코드 관리   │    │
│  │ [설치]               │    │
│  ├─────────────────────┤    │
│  │ 💬 Slack              │    │
│  │ 메시지 전송/수신      │    │
│  │ [설치됨 ✓]           │    │
│  ├─────────────────────┤    │
│  │ 🗄️ PostgreSQL        │    │
│  │ DB 쿼리 실행          │    │
│  │ [설치]               │    │
│  └─────────────────────┘    │
│                             │
│  설치된 도구 → 에이전트      │
│  인벤토리에서 장착/해제      │
└─────────────────────────────┘
```

## 흐름

1. MCP 스토어에서 도구 **검색/설치**
2. 설치된 도구가 **보관함**에 추가됨
3. 에이전트 **인벤토리**에서 장착/해제 ([inventory.md](inventory.md) 참조)
4. 장착된 에이전트가 해당 도구 **사용 가능**
